Para iniciar sesión en Qualitas Reporting de forma tradicional, se deberá especificar un nombre de usuario y su contraseña. El administrador del sistema deberá proveerle esta información. En cuanto a la configuración inicial, el sistema ya viene con un usuario prestablecido, cuyo nombre es admin y su contraseña es admin. Este usuario posee los permisos suficientes  para administrar el sistema por completo. Se recomienda estrictamente, una vez finalizada la configuración inicial, que la contraseña del usuario admin sea modificada.

En Qualitas Reporting, existe otra modalidad de inicio de sesión, que es de forma integrada a un dominio de la red Windows. De esta forma se logra la integración con Active Directory. Para ello, el administrador de seguridad deberá configurar el sistema de modo tal, indicando un nombre de dominio de forma predeterminada para que el usuario que ingrese pueda validarse utilizando su contraseña de red.

Nótese que ante un caso de emergencia, el usuario admin podrá validarse utilizando la contraseña propia de Qualitas Reporting , a no ser que dicho usuario se encuentre bloqueado. Por otra parte, no se permitirá una contraseña en blanco para el inicio de sesión.

Otra situación a tener en cuenta es la cantidad de intentos fallidos para el inicio de sesión. Para el caso de utilizarse la seguridad propia del sistema, el mecanismo de validación tolerará hasta 2 intentos fallidos, bloqueando al usuario en el tercer intento erróneo. En el caso de utilizar la seguridad integrada a Active Directory, la politica de bloqueo de claves que regirá será la de Windows.
