El diseñador visual de consultas, o también denominado Asistente para instrucciones SQL, le facilitará la creación del código SQL necesario para la ejecución de consultas, a través de un cuadro de diálogo en el que se exponen todos los objetos de la base de datos de la conexión utliizada, y mediante el cual se podrá hacer uso de arrastrar y soltar sin la necesidad de conocer a fondo el lenguaje SQL.

El diseñador consta de cinco elementos principales:

  1. El área de objetos que intervienen en la consulta
  2. El área visual de diseño
  3. La lista de objetos disponibles en la base de datos
  4. El área de columnas a manipular
  5. El cuadro de texto con la instrucción SQL generada.

Objetos que intervienen en la consulta

Sobre el margen izquierdo se encuentra el área de los objetos que se incluyen en la consulta. Es aquí donde se agrupan la consulta principal y las distintas subconsultas o tablas derivadas (en caso de existir), y para cada una de ellas las expresiones indicadas en el área visual de diseño. Para cada item correspondiente al nodo de Objetos, puede hacer doble clic de forma tal de asignar un ALIAS al objeto, y de este modo darle un poco más de coherencia a la lectura del código cuando trata con nombres de objetos poco amenos.

Area visual de diseño

El área visual de diseño es el espacio donde se podrán arrastrar y soltar todos los objetos de la base de datos que deban intervenir en la consulta.

Una vez que un objeto (tabla o vista) se encuentra en este área, se podrán indicar que columnas se desean mostrar e inclusive se podrá arrastrar una columna de un objeto hacia otra de otro objeto, a fin de establecer una relación entre los mismos.
Una vez conformada una relación, al hacer doble clic sobre la línea que la representa, se accede a un cuadro de diálogo donde se podrá establecer el tipo de unión que vincule a los objetos:

Es desde esta área donde también se podrá crear una subconsulta o tabla derivada, que le permitirá al diseñador ganar un espacio de trabajo nuevo donde podrá incluir otros objetos y poder relacionarlos con la consulta principal. Para ello, se deberá hacer clic con el botón derecho sobre un espacio libre del área visual de diseño, y seleccionar la opción "Agregar tabla derivada" del menú contextual.
Otra característica de este espacio de trabajo, es la posibilidad de crear consultas de UNION. Esta opción se encuentra disponible al hacer clic con el botón derecho sobre el ícono  , y luego seleccionando "Nueva subconsulta de unión" del menú contextual.

Objetos disponibles en la base de datos

La lista de objetos disponibles le indica al diseñador cuales son las tablas y vistas disponibles bajo el esquema de seguridad de la conexión actual, y que podrá arrastrar y soltar hasta el área visual de diseño.

También podrá agregar objetos a la consulta al presionar el botón Agregar objeto, ubicado en la esquina inferior izquierda del diseñador visual. De esta forma se identifican en cada ficha los diferentes tipos de objetos, como son tablas, vistas y procedimientos almacenados.

Area de columnas a manipular

Esta área permite establecer los alias de las columnas seleccionadas en el área visual de diseño, aplicar filtros prestablecidos, indicar el orden final de la consulta y establecer funciones de agregado cuando lo considere necesario.

Para cada campo incluido en la consulta, deberá indicarse la siguiente información:

  • Salida: casilla de verificación que indica si la columna se seleccionará o no.
  • Expresión: la expresión SQL que el motor de base de datos procesará. Normalmente es el nombre de la columna de una tabla o vista determinada.
  • Función de agregado: indica que la expresión deberá representarse con una función de agregado, a saber:
    • AVG: promedio de cada grupo
    • SUM: suma de cada grupo
    • COUNT: recuento de registros de cada grupo
    • MIN: valor mínimo de cada grupo
    • MAX: valor máximo de cada grupo
    • DISTINCT (combinable con todas las anteriores): hace ignorar los registros duplicados
  • Alias: una denominación alternativa de la expresión, que finalmente será el nombre de la columna en grillas y tablas dinámicas.
  • Tipo de orden: seleccionar entre ascendente, descendente o ninguno. En éste último caso, el orden de los resultados será el predeterminado del motor de base de datos.
  • Dirección: para el caso que se especifique Tipo de orden en más de una columna, este valor deberá consignarse con un número secuencial comenzando desde el 1 (uno) para indicar a la consulta que columna tomar para ordenar primero, y así sucesivamente.
  • Agrupamiento: casilla de verificación que indica que esa columna conformará un grupo de resultados para cada valor distinto que se encuentre en ella.
  • Filtro: si el Agrupamiento no fue indicado, se deberá especificar un valor que represente la condición de inclusión de registros, de acuerdo a los valores de esa columna. En caso contrario, se deberá indicar si el filtro se aplicará antes de procesar el agrupamiento ("For values") o después ("For groups").
  • O...: provee espacio para agregar más condiciones al criterio de selección de registros.

Cuadro de texto con la instrucción generada

El cuadro de texto del extremo inferior del diseñador, contiene la instrucción SQL que va siendo automáticamente generada por éste.

Si el administrador del sistema lo permite, este cuadro de texto será editable, agilizando la conformación de la instrucción final. De proceder con esta práctica, tenga en cuenta que, de acuerdo a la sintaxis específica que se utilice, en el área visual de diseño pueden no representarse los objetos intervinientes en la consulta, con lo cual la composición final de dicha instrucción queda al libre conocimiento del diseñador.
Nótese que el diseñador visual de consultas, traduce al lenguaje standard ANSI-92 SQL, por lo que cualquier instrucción específica del motor de bases de datos que se esté utilizando, no será reconocida, y por lo tanto, no podrá ser representada de forma gráfica.
