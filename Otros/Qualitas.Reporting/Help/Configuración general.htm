Las opciones generales de la aplicación controlan lo siguiente:

  • Ruta de acceso a la información local: especifica la carpeta del usuario donde se guardará la información temporal durante el uso de Qualitas Reporting, como así también la información de los últimos diseños de vista utilizados y será la ruta predeterminada para guardar los documentos que el usuario exporte. En esta ruta se guardará también todas las opciones de configuración del usuario.
    Tener en cuenta que al especificar una ruta de acceso, Qualitas Reporting asumirá que todos los usuarios que lo utilicen disponen de dicha ruta como accesible.
    El valor por omisión para esta opción es "C:\QUALITAS\"
  • Objetos a mostrar en el diseñador visual: enumeración de los objetos de la base de datos que estarán visibles en el diseñador visual de consultas en el momento del diseño. Estas opciones se pueden combinar entre tablas, vistas y procedimientos almacenados.
  • Interpretación SQL: son los valores por omisión que Qualitas Reporting inferirá al crear una nueva conexión a base de datos, relacionados a las características específicas de cada motor de base de datos. Dichas opciones pueden modificarse para cada conexión creada, por lo que lo que aquí se defina, sólo serán valores por omisión.
    • Símbolo decimal: es el símbolo decimal que se utiliza para valores numéricos decimales (valor por omisión ".").
    • Literal de cadenas: representa el caracter que se utilizará para encerrar los tipos de datos de texto al momento de conformar las instrucciones SQL.
    • Literal de fechas: idem Literal de cadenas, pero para los tipos de datos interpretados como fecha.
    • Formato de fechas: es el formato que responde a como el motor de base de datos interpreta una fecha. Se pueden utilizar los siguientes caracteres como comodín:
      • yyyy: año correspondiente a la fecha.
      • MM: mes correspondiente a la fecha.
      • dd: día correspondiente a la fecha.
      • - o /: separador entre año, mes y día

  • Otras opciones: 
    • Edición SQL habilitada: indicador que controla la escritura manual de código SQL. De no activarse esta opción, la única manera de desarrollar consultas será por medio del diseñador visual.

Presione el botón Aceptar para confirmar las modificaciones realizadas o Cancelar para descartarlas.
