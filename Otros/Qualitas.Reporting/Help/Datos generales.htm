Para comenzar con el diseño de una consulta, existen ciertos datos generales que el diseñador deberá proveer, y que en su mayoría son requeridos.

  • Nombre: representa el nombre que identificará una consulta de otras. Si bien dos o más consultas pueden tener el mismo nombre, el sistema alertará sobre esta intención, aunque de todos modos internamente son registradas con un código único. Este dato es requerido.

  • Descripción: proporciona un espacio para describir la funcionalidad de la consulta, a fin de orientar al usuario que la ejecute.

  • Categoría: ofrece la lista de categorías para asignarle una a la consulta. Este dato es requerido.

  • Conexión: indica al sistema cuál será la conexión a la base de datos que contiene la información que debe procesar la consulta. Este dato también es requerido.

  • Contraseña de ejecución: habilita el ingreso de una contraseña que todo aquél usuario que intente ejecutar la consulta deberá proveer.

  • Contraseña de diseño: habilita el ingreso de una contraseña que será requerida al momento de editar la consulta en modo diseño.

  • Modo servidor: opción que optimiza la consulta cuando se debe trabajar con grandes volúmenes de datos. De esta forma, se evita cargar todo el conjunto de datos devuelto por la consulta en memoria de forma simultánea.
